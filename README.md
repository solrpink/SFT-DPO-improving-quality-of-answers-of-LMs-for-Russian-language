# SFT-DPO-improving-quality-of-answers-of-LMs-for-Russian-language
Проект посвящен файн-тьюнингу модели gemma3-4b-it на синтетических данных на основе таксономии Блума. Таксономия Блума — это классификация учебных целей, которая разделяет мышление на 6 уровней сложности. Она помогает педагогам и методистам выстраивать программу от простого к сложному — от базового запоминания фактов до создания новых идей и критического анализа. Идея датасета, обучить модель на данных в таком порядке, как дети изучают в рамках школьной программы.

ВАЖНОЕ ПРИМЕЧАНИЕ! Большинство файлов находятся в репозитории, особенно связанные с кодом (но не все), sft+dpo точно должен в полном объеме присутствовать, также находятся большинство промптов, датасеты не находятся на данный момент в репозитории из-за нашей будущей научной работы, об этом будут коментарии дальше, однако на устной презентации, мы все покажем и расскажем более подробно, при наличии вопросов :) (спасибо за понимание)

# Фильтрация датасета
Филтрация датасета происходила на всех этапах его создания, но ключевые фильтры были на последнем этапе. Фильтрация была также автоматическая алгоритмом LLM-as-a-judge, модель судья google/gemma-3-27b-it. Инструкции проходили через:
1. Answerability — насколько на вопрос можно ответить (скор от 0 до 4, где 0 - FAIL, 4 - SUCCESS), сам фильтр можно найти в репозитории (в нем подробно описаны критерия одбора).
2. Factology - насколько вопрос является фактологически точным (скор от 0 до 4, где 0 - FAIL, 4 - SUCCESS), сам фильтр можно найти в репозитории (в нем подробно описаны критерия одбора).
3. Hint - присутствует ли leakage в вопросе, подсказки (скор от 0 до 100, 0-20: clean, 21-40: review, 41-70: fail, 71-100: fail), сам фильр можно найти в репозитории (в нем подробно описаны критерия одбора).

Промпты можно найти в файлах, которые называются test.

# Справочная информация о датасете + SFT
Есть главный датасет dataset.csv (к сожалению, мы не можем его опубликовать, так как он является частью нашей статьи, которая еще будет опубликована, так что пока данные у нас, но на демонстрации проекта, мы можем все показать :), приносим извинения), в нем есть следующие колонки: класс, Вопрос, Ответ, bloom_id, answerability_score, leakage_score, accuracy_score, answerability_verdict, hint_verdict, fact_verdict, answerability_issue_type, hint_issue_type, fact_error_type, Domain (Мы также прикрепляем в репо описание датасета в формате docx.

dataset.csv был преобразован в вид, где домены, класс, блум айди, а в последнюю очередь сложность чередовались, например:

# Как проходил DPO
Взяли часть датасета Mamuramu (500 примеров) только по естесственным наукам. 

Формат получился такой:
```json
{
    "prompt": "Если вы верите, что изменения происходят медленно и стабильно на протяжении всей жизни, вы будете придерживаться _____ стороны этого взгляда на развитие.\nA) Механизм\nB) Организм\nC) Непрерывность\nD) Стабильность",
    "chosen": "C",
    "rejected": "B"
  },
  {
    "prompt": "Комплекс пируватдегидрогеназы:\nA) находится в саркоплазме.\nB) катализирует превращение пирувата в ацетил-КоА.\nC) катализирует превращение пирувата в лактат.\nD) катализирует превращение лактата в пируват.",
    "chosen": "B",
    "rejected": "A"
  },
```

Часть rejected немного отфармотировали, чтобы они содержали еще какие-то объяснения, которых быть НЕ ДОЛЖНО для датасета Mamuramu:
```json
  {
    "prompt": "Корневое давление, создаваемое корнями растения, заставляет воду проникать в корни\nA) транслокация\nB) адгезия\nC) осмос\nD) капиллярное действие",
    "chosen": "C",
    "rejected": "A, транслокация"
  },
  {
    "prompt": "Потеря ____ считается в США самой серьезной потерей и часто вызывает чувство вины в дополнение к другим эмоциям.\nA) Супруга\nB) Родителя\nC) Ребенка\nD) Брата",
    "chosen": "C",
    "rejected": "A, Супруга"
  },
```

А затем, части rejected добавили тот же ответ, что и в chosen, но в rejected добавили объяснения, которых быть НЕ ДОЛЖНО для датасета Mamuramu:
```json
  {
    "prompt": "Термин \"старение\" относится к\nA) Повышенная уязвимость\nB) Процесс старения\nC) Потеря умственных способностей\nD) Биологическое старение",
    "chosen": "A",
    "rejected": "A. Старение — это биологический процесс, при котором с возрастом снижается способность организма к восстановлению и адаптации, что приводит к повышенной уязвимости перед болезнями и внешними факторами."
  },
{
    "prompt": "Какая из следующих причин является основной причиной политении в соматических клетках определенных организмов?\nA) Транскрипция РНК\nB) Суперспирализация хроматина\nC) Репликация хромосом без деления клетки\nD) Рекомбинация хромосом",
    "chosen": "C",
    "rejected": "C. Политения возникает в результате эндоредупликации — многократной репликации ДНК без последующего деления клетки (митоза). В результате сестринские хроматиды не расходятся, а остаются соединенными, формируя гигантские политеновые хромосомы."
  },
```
Затем добавили еще 1750 примеров из общего пула Mamuramu, структура не поменялась от последней. Обучение производилось 2 эпохи на разных beta ∈ {0.01, 0.1, 0.5}.

# Evaluation на Mamuramu
Бейзлайн и модель были протестированы на бенчмарке Mamuramu (а точнее на его части  ruMMLU). ruMMLU — это русскоязычный аналог MMLU — тест на знания в разных предметных областях в формате вопрос + 4 варианта ответа (A/B/C/D). Для оценки брался сэмпл до 1000 примеров (все предметы). Также из общего пула примеров исключались примеры, которые были взяты для дообучения. Измерялось accuracy %. 

Сравнивалось 5 моделей:
1. Base - google/gemma-3-4b-it без дообучения
2. после SFT дообучения
3. DPO β=0.01 поверх SFT (на маленьком датасете 500 примеров)
4. DPO β=0.1 поверх SFT (на датасете 2250 примеров)
5. DPO β=0.5 поверх SFT (на маленьком датасете 500 примеров)

Метод оценки: 
1. Через логиты (смотрится вероятность токенов A/B/C/D напрямую)
2. Через генерацию (модель реально генерирует ответ). Итоговый прогон — через генерацию, 5-shot промпты.
